# Template Kit Quickstart

When a new client comes in, start with this package before writing or designing anything from scratch.

## Recommended order

| Step | File to use | What to do |
| --- | --- | --- |
| 1 | `BLUE_TAPE_CLIENT_SITE_SYSTEM.md` | Review the design language, copy rules, and page architecture so the work stays in brand. |
| 2 | `AI_HANDOFF_PROMPT.md` | Paste the prompt into another AI along with the client’s site copy, photos, reviews, and logo files. |
| 3 | `clientSiteTemplate.config.ts` | Replace the demo content with the client’s real services, proof, service area, and CTA details. |
| 4 | `ClientSiteTemplate.tsx` | Use the starter component as the first homepage draft, then adapt sections as needed by trade. |

## Minimum client assets to gather

| Asset | Why it matters |
| --- | --- |
| Logo | Sets the identity anchor and helps tune the color balance |
| Phone number and CTA preference | Determines the lead path and button hierarchy |
| Service list | Shapes the homepage and supporting pages |
| Service area | Supports both trust and local search clarity |
| Real images | Makes the site feel grounded and credible |
| Reviews or proof points | Gives the homepage real trust material |

## How to keep the result from feeling generic

Do not let the AI simply re-skin a contractor template. Rewrite weak copy. Remove fluff. Favor real proof over decorative UI. Keep blue accents disciplined. If a section looks like it could belong to any startup, it needs another pass.

## Adaptation note

The starter component is intentionally opinionated, but it is not meant to force every client into the exact same homepage. It should be treated as a structured first draft. The section order can shift, imagery can become more urgent or more premium depending on trade, and proof can move earlier when trust is the main sales hurdle.
